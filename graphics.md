# 파트 3. 그래픽 파이프라인 요약

그래픽 파이프라인은 **3D 물체를 2D 화면의 픽셀로 출력하는 전체 과정**이다.

---

## 챕터 6. 모델변환과 시점변환

3D 공간에서 **물체와 카메라를 배치하는 방법**을 배운다.

- 지역 좌표 및 모델 좌표: 물체를 만들 때, 그 물체 안에서만 쓰는 기준 좌표
- 전역좌표: 전체 3D 공간에서 기준이 되는 좌표계
- 모델변환: 물체를 이동, 회전, 확대/축소하는 과정
- 시점변환: 카메라의 위치와 바라보는 방향을 정하는 과정
- 핵심: 물체와 카메라를 어떻게 놓느냐에 따라 화면에 보이는 모습이 달라진다.

```C
glRotatef(angle, x, y, z); // 물체 및 카메라 회전
gluLookAt(eyeX, eyeY, eyeZ, centerX, centerY, centerZ, upX, upY, upZ); // 카메라 위치
glPushMatrix(); // 현재 변환 상태를 저장
glPopMatrix(); // 저장해둔 상태 복구
glLoadIdentity(); // 현재 선택된 행렬을 “처음 상태”로 초기화하는 명령
gluPerspective(fovy, aspect , near, far);
```

## 챕터 7. 투상 변환과 뷰포트 변환

3D 장면을 **2D 화면 좌표로 바꾸는 과정**을 배운다.

- 투상 변환: 3D 물체를 2D 평면에 나타내는 과정
- 직교 투상: 멀리 있어도 크기가 변하지 않음
- 축측투상 = 물체를 돌림
- 경사투상 = 앞면은 그대로, 깊이만 비스듬히ㄴ
- 원근 투상: 멀리 있는 물체가 작게 보임
- 정규화 변환 = 좌표를 -1부터 1까지의 표준 공간으로 맞추는 것
- 뷰포트 변환: 정규화된 좌표를 실제 화면 픽셀 좌표로 바꿈
- 정규화 가시 부피 = 투상 변환 후, 보이는 영역을 -1~1 범위의 표준 3D 공간으로 바꾼 것

* 원근분할?
원근 분할은 투상 변환 후 동차좌표 (x, y, z, w)를 w로 나누어 정규화 장치 좌표로 변환하고, 원근감을 만들어내는 과정이다.

glOrtho()는 직교 투상을 설정하는 OpenGL 함수로, left, right, bottom, top, near, far 값으로 직육면체 형태의 가시 부피를 정의한다. 직교 투상에서는 원근감이 없으므로 물체의 크기가 거리와 관계없이 일정하게 나타난다.

1단계: 투상 변환
2단계: 정규화 / 원근 분할
3단계: 뷰포트 변환

## 챕터 8. 가시성판단

화면에서 **보이는 부분과 가려지는 부분을 판단하는 방법**을 배운다.

후면제거: 카메라에서 보이지 않는 물체의 뒷면은 아예 그리지 않는  방법
ㄴ 면의 법선 벡터가 카메라를 향하지 않으면 그 면을 제거한다.

- 가까운 물체는 보이고, 뒤에 있는 물체는 가려져야 한다.
- 대표 방법: Z-buffer 알고리즘
- 핵심: 어떤 물체를 최종 화면에 그릴지 결정한다.


### 절단 알고리즘

코헨-서더런드 = 선분 + 지역 코드
ㄴ경우 1. 둘 다 0000이면 완전히 보임
ㄴ경우 2. AND 결과가 0이 아니면 완전히 안 보임
ㄴ경우 3. 둘 다 아니면 일부만 보임

리앙-바스키 = 선분 + 매개변수 t
ㄴ선분이 창에 들어오는 t값과 나가는 t값을 계산 → 그 사이만 남김

서더런드-핫지먼 = 다각형 + 경계별로 자르기
ㄴ서더런드-핫지먼 알고리즘은 다각형 절단 알고리즘으로, 절단 창의 각 경계에 대해 다각형의 모든 변을 순서대로 검사하며 내부와 외부를 판별한다. 각 변의 시작점과 끝점이 내부인지 외부인지에 따라 끝점 또는 교점을 결과 정점 리스트에 추가하고, 이 과정을 모든 절단 경계에 대해 반복하여 최종 절단 다각형을 얻는다.

웨일러-애서톤 = 다각형 + 복잡한 경계 추적
ㄴ웨일러-애서톤 알고리즘은 다각형 절단 알고리즘으로, 다각형과 절단 창의 교차점을 찾고 진입점과 진출점으로 구분한 뒤, 다각형 경계와 절단 창 경계를 번갈아 따라가며 절단 결과 다각형을 생성한다.


물체 공간 알고리즘: 물체나 면끼리 직접 비교해서 어떤 면이 보이는지 판단하는 방법
영상 공간 알고리즘: 화면의 픽셀마다 어떤 물체가 가장 앞에 있는지 판단하는 방법

1) Z-buffer 알고리즘

각 픽셀마다 깊이값 z를 저장해서, 가장 가까운 픽셀만 화면에 그리는 방법

2) 깊이 정렬 알고리즘(페인터)

멀리 있는 물체부터 먼저 그리고, 가까운 물체를 나중에 덮어 그리는 방식이다

3) 주사선 알고리즘

화면을 한 줄씩, 즉 scan line 단위로 검사하면서 어떤 면이 앞에 있는지 판단하는 방법

## 챕터 9. 래스터변환

선, 삼각형, 다각형 같은 도형을 **픽셀로 바꾸는 과정**을 배운다.

- 화면은 픽셀로 이루어져 있기 때문에 도형을 픽셀 단위로 변환해야 한다.
- 선 그리기, 원 그리기, 다각형 채우기 등을 다룬다.
- 핵심: 수학적 도형을 실제 화면 픽셀로 표현한다.

### 선분 래스터 변환

선을 픽셀로 표현하는 방법을 배운다.


교차점 계산 방식: 직선 방정식으로 y값을 계산하고 반올림해서 픽셀 선택, 매번 직선 방정식에 x를 넣어서 y를 새로 구하는 방식

DDA 알고리즘: x 또는 y를 1씩 증가시키며, 다른 좌표를 기울기에 따라 계산해 픽셀을 선택하는 알고리즘

브레즈넘 알고리즘: 정수 연산으로 다음 픽셀을 빠르게 선택, 결정변수 D사용

- 주사선 채움 알고리즘
ㄴ주사선 채움은 다각형을 가로줄 단위로 채우는 방법

- 특수 경우 처리
ㄴ꼭짓점이나 경계에서 예외 처리가 필요하다

씨앗 채움 알고리즘
= 내부의 한 점에서 시작해서 주변 픽셀로 퍼지며 영역을 채우는 방법

보간법
= 두 값 사이의 중간값을 비율에 따라 계산하는 방법

## 챕터 10. 조명과 음영

물체가 **입체감 있고 현실적으로 보이도록 빛과 밝기를 계산하는 방법**을 배운다.

- 조명: 빛이 물체에 닿았을 때의 밝기와 색을 계산
- 주변광, 난반사광, 정반사광을 다룬다.
- 음영: 물체 표면의 밝기를 부드럽게 표현하는 방법
- 플랫 쉐이딩: 면 하나를 한 가지 밝기로 칠하는 방식
- 구로 쉐이딩: 꼭짓점에서 밝기를 계산하고, 그 사이를 보간해서 칠하는 방식
- 퐁 쉐이딩: 꼭짓점의 법선 벡터를 보간하고, 각 픽셀마다 조명을 계산하는 방식


플랫 쉐이딩: 면 단위로 같은 색을 칠함. 빠르지만 각져 보인다.
구로 쉐이딩: 정점의 밝기를 보간함. 비교적 부드럽지만 하이라이트 표현이 약하다.
퐁 쉐이딩: 법선을 보간하고 픽셀마다 조명 계산함. 가장 자연스럽지만 계산량이 많다.
